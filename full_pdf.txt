1. Overview
This document provides a complete reference for the HERO Software GraphQL API. The API allows external
integrations to read and write CRM data including contacts, projects, calendar events, tasks, documents,
products, time tracking, and more.

2. Authentication
All API requests require a Bearer token in the Authorization header. Generate your API token in HERO
Software under Settings > API Access.

curl -X POST \​
https://login.hero-software.de/api/external/v9/graphql \​
-H "Authorization: Bearer YOUR_TOKEN" \​
-H "Content-Type: application/json" \​
-d '{"query": "{ user { id email } }"}'

The `Float` scalar type represents signed double-precision

The `ID` scalar type represents a unique identifier, often used
to​
refetch an object or as key for a cache. The ID type appears in a
JSON​
response as a String; however, it is not intended to be
human-readable.​
When expected as an input type, any string (such as `"4"`) or
integer​
(such as `4`) input value will be accepted as an ID.
The `Int` scalar type represents non-fractional signed whole
numeric​
values. Int can represent values between -(2^31) and 2^31 - 1.
